# Guía rápida del administrador

## Antes de cambiar información

- Confirme ambiente, sistema y cuenta `ADMIN`.
- Confirme quién solicita y quién autoriza.
- Revise los datos dos veces.
- Registre el motivo.
- No copie claves ni sesiones de acceso.

## Datos maestros

### Artículo

- Código único, en mayúsculas.
- Solo letras, números, `-` y `_`.
- Descripción clara.
- No puede editarse o eliminarse con el prototipo.

### Ubicación

- Formato como `A-01-01`.
- Código y secuencia únicos.
- Comprobar físicamente antes de crear.
- No puede editarse o eliminarse con el prototipo.

## Ajuste de existencias

1. Realice un segundo conteo.
2. Compruebe artículo y ubicación.
3. Revise tareas abiertas y terminales con `Q:`.
4. Use `+` para aumentar y `-` para reducir.
5. Escriba una razón concreta.
6. Compruebe la cantidad resultante y registre el movimiento.

**Si la respuesta es dudosa, no repita el ajuste.** Soporte debe comprobar primero
si ya fue aplicado.

## Crear una orden

- Número único.
- Líneas distintas.
- Artículos existentes.
- Cantidades mayores que cero.
- Existencias disponibles suficientes.
- Segunda revisión antes de enviar.

La orden se crea completa o no se crea. El prototipo no permite editarla ni
cancelarla.

## Bloquear o reanudar

- Actúe solo con solicitud autorizada.
- Antes de bloquear, compruebe si la terminal muestra `Q:`.
- Use una razón detallada.
- Bloquear y reanudar no cambian existencias.
- Después de reanudar, compruebe `AVAILABLE`.

## Etiquetas

- Use el código exacto existente.
- Prefiera PDF para imprimir.
- No recorte el QR.
- Compare el texto y escanee una muestra.

## Nunca haga esto

- No improvise códigos para evitar un duplicado.
- No ajuste existencias sin conteo e historial.
- No repita una acción después de una respuesta incierta.
- No cambie información directamente en la base de datos.
- No comparta cuentas administrativas.
