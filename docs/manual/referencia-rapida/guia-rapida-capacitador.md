# Guía rápida del capacitador

## Antes

1. Confirme perfil y versión del sistema/manual.
2. Use desarrollo y datos de demostración.
3. Compruebe terminal, red, batería, cuentas y etiquetas.
4. Prepare camino correcto y al menos un problema.
5. Pregunte de forma privada si alguien necesita apoyo adicional.

## Durante

1. Explique el propósito y los límites del puesto.
2. Muestre una vez el flujo completo.
3. Permita práctica acompañada.
4. Permita práctica sin ayuda usando el manual.
5. Evalúe decisiones seguras, no rapidez ni memoria del inglés.
6. No pida ni observe claves personales.

```text
Mostrar → practicar → explicar con sus propias palabras
```

## Si ocurre un error

- Detenga solo si existe riesgo para datos o trabajo físico.
- Pregunte qué esperaba ver.
- Vuelva al paso escrito.
- Permita repetir desde un estado seguro.
- Registre si falló el manual, sistema, ambiente o práctica.

## Para aprobar

- Todos los puntos críticos deben estar demostrados.
- Asistir no equivale a aprobar la práctica.
- Una guía rápida no sustituye el manual completo.
- Un punto crítico pendiente requiere nueva práctica.

## Después

1. Registre participantes, versiones y resultado.
2. Cree incidencias para instrucciones confusas o defectos.
3. Programe nueva práctica cuando sea necesaria.
4. Entregue la guía rápida correspondiente.
5. Conserve evidencia sin secretos ni datos personales innecesarios.

Use el [registro de capacitación](../plantillas/registro-de-capacitacion.md) y la
[validación con usuario](../plantillas/validacion-de-instrucciones-con-usuario.md).
